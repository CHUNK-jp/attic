Features

なぜ Attic なのか

クラウドに送らない。学習されない。あなたのファイルはあなたのものだけ。

🔒

完全ローカル処理

データは一切外部に送信されません。埋め込みモデルも検索も、すべてあなたのマシン上で完結します。

🧠

自然言語検索

ファイル名を覚えていなくても大丈夫。「Q1の予算報告」「田中さんとの会議メモ」など意味で検索できます。

⚡

高速ベクトル検索

ChromaDB によるベクトルインデックスで、数百件のドキュメントも瞬時に意味検索。

📁

多様なファイル対応

PDF・Word・テキスト・Markdown・コードなど、日常使うほぼすべての形式に対応。

🔄

差分インデックス

変更されたファイルだけを再インデックス。大量のファイルがあっても2回目以降は高速です。

🆓

無料・オープンソース

MIT ライセンス。サブスクリプションなし、API キーなし、登録不要。今すぐ始められます。

How It Works

仕組み

ファイルをチャンクに分割し、Ollama で埋め込みベクトルに変換。ChromaDB に保存して高速検索します。

関連ファイルを表示

nomic-embed-text

Ollama 経由でローカル実行される埋め込みモデル。高品質な多言語対応ベクトルを生成します。

ChromaDB

オープンソースのベクトルデータベース。インデックスは ~/.attic/db にローカル保存されます。

差分更新

ファイルのハッシュを管理し、変更分だけを再インデックス。効率的に最新状態を維持します。

Quick Start

4ステップで始める

Python 3.10 以上と Ollama が必要です。

1

Ollama をインストールして埋め込みモデルを取得

ollama.com からインストール後、モデルを pull します。

ollama pull nomic-embed-text
2

リポジトリをクローンして依存関係をインストール

git clone https://github.com/CHUNK-jp/attic.git && cd attic
pip install -r requirements.txt
3

フォルダをインデックス

python3 attic.py index ~/Documents
4

自然言語で検索

python3 attic.py search "Q1 budget report"
